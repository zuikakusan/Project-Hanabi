# 系统需求表

## 1. 优先级定义

| 优先级 | 定义                    |
| --- | --------------------- |
| P0  | Demo 必须实现，没有它核心流程不能成立 |
| P1  | Demo 可选增强，能显著改善体验     |
| P2  | 正式版或后续版本内容，当前不进入 Demo |

## 2. P0 必须系统

### 2.1 角色控制

| 功能 | 说明 | 验收 |
|---|---|---|
| 移动 | WASD / 手柄摇杆移动 | 玩家能在所有场景自由移动 |
| 转向 | 朝移动方向或瞄准方向转身 | 角色朝向稳定，不漂移 |
| 冲刺 | 短时间加速 | 可用于躲避敌人 |
| 基础动画 | 待机、走、跑、受击、交互 | 动作状态能正确切换 |

推荐：固定上帝视角 / 斜俯视第三人称镜头，不是越肩视角。优先保证玩家能看见周围环境、空间结构和投掷落点。

### 2.2 镜头系统

| 功能 | 说明 | 验收 |
|---|---|---|
| 半俯视跟随 | 镜头跟随玩家 | 玩家始终可见 |
| 非越肩视角 | 镜头不贴近角色肩后 | 能看见玩家周围环境 |
| 固定角度 | 玩家不能旋转镜头 | 镜头角度始终稳定 |
| 战斗视野 | 战斗时能看清敌人和弹道 | 爆炸和敌人不频繁出屏 |

### 2.3 交互系统

统一所有可交互对象：

- NPC
- 可调查物体
- 道具拾取
- 门
- 场景入口
- 商店
- 小游戏触发点

| 功能 | 说明 | 验收 |
|---|---|---|
| 交互范围 | 玩家靠近后触发 | 出现提示 |
| 交互提示 | 显示按键和对象名称 | 提示不遮挡画面 |
| 触发事件 | 按键后执行对应逻辑 | 对话、拾取、切换场景正常 |

### 2.4 对话系统

| 功能 | 说明 | 验收 |
|---|---|---|
| 显示说话人 | NPC 名称和文本 | 文本可读 |
| 多句推进 | 按键推进句子 | 不跳句 |
| 选项 | 2 到 4 个选项 | 可选择并记录结果 |
| 变量判断 | 根据任务、道具、阶段显示不同文本 | 王小虎胜负后文本不同 |
| 触发事件 | 对话可给道具、开任务、开小游戏 | 主流程能串联 |

### 2.5 任务系统

| 功能 | 说明 | 验收 |
|---|---|---|
| 接取任务 | 由对话或场景触发 | 任务进入列表 |
| 更新目标 | 阶段推进 | UI 显示当前目标 |
| 完成任务 | 达成条件后完成 | 触发奖励或下一任务 |
| 变量记录 | 保存关键状态 | 退出场景后状态不丢 |

Demo 主任务：[[02 Demo流程文档 - 42号楼#5.1 主任务：42号楼的名字|42号楼的名字]]。

### 2.6 背包系统

| 功能 | 说明 | 验收 |
|---|---|---|
| 获得道具 | 对话、拾取、商店给予 | 道具进入背包 |
| 查看道具 | 简单列表或网格 | 显示名称、数量、描述 |
| 消耗道具 | 使用爆竹时扣除数量 | 数量正确变化 |
| 任务物品 | 不可丢弃 | 名册碎页保留 |

### 2.7 武器系统

| 功能 | 说明 | 验收 |
|---|---|---|
| 武器切换 | 四种爆竹切换 | UI 显示当前武器 |
| 弹药消耗 | 使用对应道具 | 数量减少 |
| 投射物生成 | 摔炮、擦炮、窜天猴 | 弹道正确 |
| 持续发射 | 魔术棒持续一段时间 | 每秒发射小弹药 |
| 爆炸判定 | 范围伤害 | 敌人正确受击 |
| 噪音事件 | 大声音吸引敌人 | 窜天猴能吸引保安幻影 |
| 冷却 | 防止连续无脑使用 | UI 或手感反馈明确 |

初始数值见 [[04 内容表#5. 武器表]]。

### 2.8 敌人系统

| 功能 | 说明 | 验收 |
|---|---|---|
| 待机 | 敌人默认状态 | 不乱跑 |
| 发现玩家 | 距离或视野触发 | 转入追击 |
| 声音吸引 | 听到爆竹声前往位置 | 能被窜天猴吸引 |
| 追击 | 向玩家移动 | 路径基本可用 |
| 攻击 | 接近后扣勇气值 | 有冷却 |
| 受击 | 被爆炸影响 | 播放反馈 |
| 出局 | 生命归零消失或倒地 | 不再攻击 |

### 2.9 场景切换

| 功能 | 说明 | 验收 |
|---|---|---|
| 日常到异常 | 后巷进入神秘街道 | 场景可切换 |
| 日常到秘境 | 进入 42号楼 | 玩家状态保留 |
| 秘境回日常 | Boss 后回到学校门口 | 任务继续 |

### 2.10 小刀一把小游戏

| 功能 | 说明 | 验收 |
|---|---|---|
| 猜拳 | 双方出拳，判定胜负 | 胜者获得行动 |
| 位置 | 自家阵地、中间、对方阵地 | UI 清楚 |
| 道具 | 拿纸刀、抢道具 | 状态正确 |
| 行动 | 前进、出击、防守、抢道具、嘴硬 | 能影响胜负 |
| 胜负 | 一方出局结束 | 正确回到主流程 |
| 失败兜底 | 失败可重试或走备用线索 | 不阻塞 Demo |

### 2.11 UI 系统

P0 需要的 UI：

- 交互提示
- 对话框
- 任务目标
- 背包
- 当前武器和弹药
- 勇气值 / 状态值
- 小游戏界面
- 场景切换提示

## 3. P1 增强系统

| 系统 | 功能 | 价值 |
|---|---|---|
| 时间系统 | 早晨、课间、放学、黄昏、夜晚 | 强化 P5 式日常结构 |
| 商店系统 | 购买、出售、特殊商品 | 让老孟商店更完整 |
| NPC 关系 | 好感、敌对、面子 | 支撑儿童江湖 |
| 属性系统 | 胆量、机灵、面子、幻想力 | 支撑文本和成长 |
| 内心声音 | 根据属性弹出提示 | 增加《极乐迪斯科》式文本厚度 |
| 秘境机关 | 门、钥匙、简单谜题 | 提升探索感 |
| 存档系统 | 保存任务和背包 | 适合更长 Demo |
| 音频系统 | 环境声、铃声、爆竹声、远处电视声 | 强化怀旧和恐惧 |

## 4. P2 后续系统

| 系统 | 说明 |
|---|---|
| 多主角 | 不同区域、不同家庭、不同初始能力 |
| 多区域城市 | 老城区、农村、城中村、新城区 |
| 多小游戏 | 弹珠、圆牌、躲猫猫、扔沙包等 |
| 组织系统 | 分会、地盘、声望、任务 |
| 队友系统 | 小队协作探索 |
| 高级敌人 AI | 巡逻、听觉、群体围攻、恐惧反应 |
| 道具制作 | 爆竹改造、玩具改造 |
| 多结局 | 根据组织关系和真相探索程度变化 |
| 家庭系统 | 作业压力、父母关系、夜晚限制 |

## 6. 制作任务拆分

第一批建议任务：

1. 主角移动和镜头。
2. 基础交互系统。
3. NPC 对话系统。
4. 任务目标系统。
5. 背包和道具拾取。
6. 四种爆竹武器原型。
7. 敌人基础 AI。
8. 小型秘境战斗房间。
9. 小刀一把小游戏原型。
10. Demo 任务流程串联。
11. 结尾剧情触发。
