# 核心射击系统方向研究

## 1. 文档目的

这份文档用于确定《千禧年战纪》核心射击系统的方向。

它不是完整战斗系统规格，也不是立即交付实现的任务单。当前目标是把以下关键方向先定下来：

- 视角类型
- 基础移动与瞄准方式
- 爆竹武器的操作逻辑
- 点火、投掷、延迟爆炸的核心体验
- 第一版战斗原型应该验证什么
- 当前阶段不做什么

## 2. 当前结论

### 2.1 视角结论

游戏采用固定第三人称视角，但这里的“第三人称”不是越肩视角，也不是传统肩后 TPS。镜头应参考《孤胆枪手》的上帝视角 / 俯视角射击体验，能看见玩家角色和周围环境。

当前方向：

| 项目 | 结论 |
|---|---|
| 视角类型 | 固定上帝视角 / 斜俯视第三人称 |
| 参考对象 | 《孤胆枪手》 |
| 战斗形态 | 俯视 / 半俯视 TPS |
| 是否越肩 | 不是越肩视角 |
| 可见范围 | 能看见角色周围环境 |
| 移动方式 | 键盘移动 |
| 瞄准方式 | 鼠标瞄准 |
| 核心攻击方式 | 爆竹、烟花、玩具化武器 |

这里的 TPS 不是传统肩后枪战，而是更偏“俯视角第三人称动作射击”。玩家要能观察周围敌人、爆炸范围、障碍物和撤退路线。

### 2.2 输入结论

基础输入：

| 输入 | 功能 |
|---|---|
| W / A / S / D | 角色移动 |
| 鼠标移动 | 控制瞄准方向，同时控制人物朝向 |
| 鼠标右键 | 点火 |
| 鼠标左键 | 投掷 / 发射 |

人物朝向规则：

- 人物朝向由鼠标控制。
- WASD 只控制移动，不直接决定人物朝向。
- 玩家可以边后退、侧移，边朝鼠标方向瞄准。
- 投掷方向使用人物当前朝向。

## 3. 核心战斗体验

射击系统的核心不是“枪法”，而是“点火、预判、投掷、爆炸、转移”的节奏。

目标体验：

```text
发现敌人 -> 选择爆竹 -> 点火 -> 瞄准 -> 投掷 / 发射 -> 等待爆炸 -> 观察反馈 -> 转移位置
```

与普通枪战的区别：

| 普通枪战 | 《千禧年战纪》爆竹战斗 |
|---|---|
| 点击立即射击 | 很多武器需要先点火 |
| 子弹即时命中 | 爆炸有延迟 |
| 主要考验瞄准精度 | 同时考验点火时机、投掷预判和距离控制 |
| 枪械稳定 | 爆竹可能失败、延迟、偏移、暴露位置 |
| 弹药抽象 | 武器本身是可消耗道具 |

## 4. 点火系统

### 4.1 点火是战斗的核心差异

点火系统是本作爆竹战斗和普通射击最大的区别。

玩家不是简单按左键攻击，而是需要在某些武器上经历：

```text
选择武器 -> 点火 -> 进入已点燃状态 -> 投掷 / 发射 -> 延迟爆炸或持续发射
```

### 4.2 点火方式

目前确定两种点火方式：

| 点火方式 | 成功率 | 特点 | 风险 |
|---|---:|---|---|
| 打火机点火 | 必定成功 | 稳定、可靠 | 可能后续设计为消耗时间或暴露位置 |
| 盒子摩擦片点火 | 高概率失败 | 更有童年质感，体现不稳定 | 需要多次尝试，影响战斗节奏 |

### 4.3 输入建议

鼠标右键负责点火。

点火时需要根据当前武器决定行为：

| 当前武器 | 右键行为 |
|---|---|
| 擦炮 | 尝试点火 |
| 摔炮 | 无需点火，可不响应或提示“摔炮不用点。” |
| 魔术棒 | 点火后进入持续发射状态 |
| 窜天猴 | 后续待定，可能需要点火后发射 |

### 4.4 点火状态

建议至少有以下状态：

| 状态 | 说明 |
|---|---|
| 未点火 | 武器尚未激活 |
| 点火中 | 正在尝试点火 |
| 点火失败 | 本次尝试失败，可再次右键 |
| 已点燃 | 武器已进入危险状态 |
| 已投掷 / 已发射 | 武器离开玩家 |
| 已爆炸 / 已结束 | 武器效果结算完毕 |

## 5. 武器一：擦炮

### 5.1 设计定位

擦炮是第一种具有“点火 + 延迟爆炸”完整链路的爆竹武器。

定位：

- 投掷型
- 延迟爆炸
- 中等范围
- 需要提前预判
- 能体现爆竹战斗特色

### 5.2 操作流程

```text
选择擦炮
  -> 右键点火
  -> 点火成功
  -> 爆炸倒计时开始，5 秒
  -> 鼠标瞄准方向
  -> 左键投掷
  -> 擦炮沿抛物线飞出
  -> 倒计时结束
  -> 爆炸
```

### 5.3 关键规则

| 项目 | 当前规则 |
|---|---|
| 是否需要点火 | 需要 |
| 点火输入 | 鼠标右键 |
| 投掷输入 | 鼠标左键 |
| 爆炸计时起点 | 点火成功时 |
| 爆炸延迟 | 5 秒 |
| 投掷方式 | 朝鼠标瞄准方向投掷 |
| 弹道 | 抛物线 |
| 未点火能否投掷 | 待定，建议第一版不允许 |

### 5.4 设计风险

擦炮的关键风险是：爆炸计时从点火开始，而不是从落地开始。

这会产生很有意思的玩法：

- 点火后立刻扔，敌人有更多时间躲。
- 点火后等一会儿再扔，可以空中或落地后更快爆炸。
- 等太久可能来不及扔，甚至在玩家附近爆炸。

这非常有爆竹感，但第一版原型需要决定：

| 问题 | 当前建议 |
|---|---|
| 已点燃擦炮能否一直拿在手里 | 可以短时间拿，但爆炸倒计时继续 |
| 倒计时结束还没扔会怎样 | 原型阶段建议在玩家当前位置爆炸或强制掉落 |
| 是否伤害玩家 | 第一版可先不伤害玩家，只做视觉和范围反馈 |

## 6. 武器二：摔炮

### 6.1 设计定位

摔炮是最简单的投掷爆炸武器，用于验证基础投掷和即时爆炸。

定位：

- 不需要点火
- 左键直接投掷
- 落地立刻爆炸
- 伤害较低
- 范围较小
- 操作最简单

### 6.2 操作流程

```text
选择摔炮
  -> 鼠标瞄准方向
  -> 左键投掷
  -> 摔炮抛出
  -> 落地立刻爆炸
```

### 6.3 关键规则

| 项目 | 当前规则 |
|---|---|
| 是否需要点火 | 不需要 |
| 点火输入 | 无 |
| 投掷输入 | 鼠标左键 |
| 爆炸计时 | 落地立刻爆炸 |
| 弹道 | 抛物线 |
| 失败率 | 无 |
| 设计用途 | 第一版战斗原型最稳的起点 |

### 6.4 右键处理

摔炮无需点火。玩家右键时可以：

| 方案 | 说明 | 推荐 |
|---|---|---|
| 无反应 | 简单，但反馈弱 | 可用 |
| 显示提示“摔炮不用点。” | 反馈清楚 | 推荐 |
| 播放失败动作 | 稍复杂 | 后续 |

## 7. 武器三：魔术棒

### 7.1 设计定位

魔术棒是持续发射型武器。

定位：

- 需要点火
- 点火后自动持续发射
- 玩家主要负责移动和瞄准方向
- 输出稳定但单发伤害低

### 7.2 操作流程

```text
选择魔术棒
  -> 右键点火
  -> 点火成功
  -> 魔术棒进入持续发射状态
  -> 玩家用鼠标控制发射方向
  -> 系统按固定频率自动射出小弹
  -> 持续时间结束
```

### 7.3 关键规则

| 项目 | 当前规则 |
|---|---|
| 是否需要点火 | 需要 |
| 点火输入 | 鼠标右键 |
| 发射输入 | 点火后自动发射 |
| 瞄准方式 | 鼠标方向 |
| 发射频率 | 初始建议每 1 秒 1 发 |
| 持续时间 | 初始建议 60 秒，原型可缩短 |
| 单发伤害 | 低 |
| 用途 | 稳定输出、压制 |

### 7.4 原型建议

第一版战斗原型不建议先做 60 秒完整持续时间。

建议：

- 原型版持续 10 到 15 秒。
- 先验证自动发射、瞄准和命中反馈。
- 正式数值以后再拉长。

## 8. 武器四：窜天猴

### 8.1 当前状态

窜天猴方向已在上层设计中确定为直线远程爆炸武器，但本次输入细节尚未完全定死。

已知定位：

- 直线飞行
- 命中爆炸
- 声音大
- 容易暴露位置
- 高爆发

### 8.2 待定问题

| 问题 | 候选方向 |
|---|---|
| 是否需要右键点火 | 需要点火后左键发射 / 左键直接发射 |
| 是否有偏航 | 有轻微偏航更有烟花感 |
| 是否碰撞即爆 | 是 |
| 是否飞到最大距离后爆炸 | 可选 |
| 是否吸引敌人 | 是，但敌人系统后续再做 |

窜天猴暂时不作为第一版战斗原型核心。

## 9. 第一版战斗原型建议

### 9.1 不要直接塞进废弃小屋 Demo

当前废弃小屋 Demo 的目标是验证探索和互动，不建议直接加入射击。

建议单独做一个：

> 爆竹射击白盒测试场

### 9.2 第一版原型目标

第一版战斗原型只验证：

- 固定上帝视角 / 斜俯视第三人称是否适合射击。
- WASD 移动 + 鼠标瞄准是否舒服。
- 左键投掷是否清楚。
- 右键点火是否清楚。
- 摔炮落地即爆是否成立。
- 擦炮点火后 5 秒爆炸是否有趣。
- 爆炸范围是否可读。
- 假人 / 靶子受击反馈是否清楚。

### 9.3 第一版只做两种武器

推荐顺序：

| 顺序 | 武器 | 原因 |
|---:|---|---|
| 1 | 摔炮 | 最简单，验证投掷、落地、爆炸、命中 |
| 2 | 擦炮 | 验证点火、延迟爆炸、预判玩法 |

魔术棒和窜天猴先不进第一版战斗原型，避免范围变大。

## 10. 爆竹射击白盒测试场

### 10.1 地图

测试场只需要一个小空地。

内容：

- 玩家出生点
- 几个静止靶子
- 几个简单障碍物
- 武器切换或固定武器测试点
- 清晰边界

### 10.2 靶子

第一版不需要敌人 AI。

靶子要求：

- 可以被爆炸命中。
- 命中后有反馈。
- 可以显示受击或消失。
- 不需要移动。
- 不需要攻击玩家。

### 10.3 验证路线

```text
进入测试场
  -> 选择摔炮
  -> 鼠标瞄准
  -> 左键投掷
  -> 落地爆炸
  -> 靶子反馈
  -> 选择擦炮
  -> 右键点火
  -> 左键投掷
  -> 5 秒后爆炸
  -> 靶子反馈
```

## 11. 当前不做内容

战斗第一版不做：

- 真实敌人 AI
- 敌人攻击玩家
- 生命值系统
- 玩家受伤
- 弹药背包
- 商店补给
- 武器升级
- 四种武器全做
- 噪音吸引系统
- 队友系统
- 战斗关卡
- Boss
- 掉落奖励

先验证手感，不做完整战斗。

## 12. 需要后续决策的问题

| 问题 | 当前建议 |
|---|---|
| 擦炮点燃后是否会伤害玩家 | 第一版不伤害，只做爆炸反馈 |
| 擦炮点燃后能否取消 | 第一版不能取消 |
| 擦炮没扔出去倒计时结束怎么办 | 在玩家当前位置爆炸或强制掉落，需后续确认 |
| 点火失败率是多少 | 摩擦片点火可先设 30% 失败率 |
| 打火机点火是否有代价 | 第一版无代价，后续可增加时间或资源限制 |
| 摔炮右键是否提示 | 建议提示“摔炮不用点。” |
| 魔术棒原型持续时间 | 原型 10 到 15 秒，正式版再拉长 |
| 窜天猴是否进入第一版战斗原型 | 暂不进入 |
