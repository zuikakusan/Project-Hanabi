# 最小 Demo 地图白盒设计

## 1. 文档目的

这份文档用于指导实现者搭建第一版废弃小屋 Demo 的白盒地图。

目标不是做最终美术，而是把以下内容先搭清楚：

- 房间 B、房间 A、户外区域的空间关系。
- 玩家出生点、纸条、门 B、柜子、门 A、终点触发区的位置。
- 玩家从出生到结束的路线。
- 墙体、门、柜子、终点区域的阻挡和触发规则。

关联文档：[[06 最小Demo设计文档 - 废弃小屋]]

## 2. 白盒设计原则

第一版地图要遵循几个原则：

1. **路线清楚**：玩家不需要猜地图结构，应该自然从房间 B 走向房间 A，再走向户外。
2. **互动对象显眼**：纸条、门、柜子不能藏得太深，第一版主要验证互动流程。
3. **空间很小**：完整流程应在 1 到 3 分钟内完成。
4. **允许试错**：玩家可以先去摸门 A，看到“门上锁了”，再去找柜子。
5. **不做复杂迷宫**：这不是解谜关卡，只是“锁门 + 找钥匙”的最小探索链。

## 3. 地图整体结构

地图由三个主要区域组成：

```text
房间 B：出生点、纸条 X、门 B
    ↓
房间 A：门 B 入口、门 A、柜子 / 钥匙
    ↓
户外区域：出口、终点触发区
```

推荐采用纵向串联结构，便于俯视角阅读：

```text
┌────────────────────────────┐
│            房间 B           │
│                            │
│    P                       │
│                            │
│          X                 │
│                            │
│                      DB    │
└──────────────────────┬─────┘
                       │
┌──────────────────────┴─────┐
│            房间 A           │
│                            │
│    C                       │
│                            │
│                      DA    │
└──────────────────────┬─────┘
                       │
              户外区域 / END
```

图例：

| 标记 | 含义 |
|---|---|
| P | 玩家出生点 |
| X | 纸条 X |
| DB | 门 B |
| C | 柜子，内含门 A 钥匙 |
| DA | 门 A |
| END | 户外终点触发区 |

## 4. 推荐尺寸

这里的尺寸是白盒建议，单位可以按 Godot 中的世界单位理解。实现者可以按项目单位等比例调整。

| 区域   |      推荐尺寸 | 说明                |
| ---- | --------: | ----------------- |
| 房间 B |    10 x 8 | 起点房间，容纳出生点、纸条、门 B |
| 房间 A |    10 x 7 | 过渡房间，容纳柜子和门 A     |
| 户外区域 |    10 x 5 | 只作为出口和 Demo 结束区域  |
| 墙体厚度 | 0.3 到 0.5 | 只要能阻挡玩家即可         |
| 门宽   |   1.5 到 2 | 需要明显可通过           |
| 互动半径 |   1.5 到 2 | 玩家靠近后出现 E 提示      |

### 4.1 尺寸要求

- 玩家从出生点走到纸条：约 2 到 4 秒。
- 玩家从纸条走到门 B：约 3 到 6 秒。
- 玩家从门 B 走到门 A：约 3 到 6 秒。
- 玩家从门 A 走到柜子：约 3 到 6 秒。
- 玩家从门 A 走到户外终点：约 3 到 6 秒。

总流程不应因为空间过大而拖慢。

## 5. 坐标建议

为了方便实现者搭建，下面给出一套简单坐标方案。地图中心可以自行调整。

假设使用俯视图坐标：

- X 轴：左右方向。
- Z 轴：上下方向。
- 房间 B 在上方。
- 房间 A 在中间。
- 户外区域在下方。

| 对象        |           推荐坐标 | 朝向 / 备注             |     |
| --------- | -------------: | ------------------- | --- |
| 房间 B 中心   |      (0, 0, 8) | 起点房间                |     |
| 房间 A 中心   |      (0, 0, 0) | 过渡房间                |     |
| 户外区域中心    |     (0, 0, -6) | 终点区域                |     |
| 玩家出生点 P   |    (-3, 0, 10) | 面向房间中央或纸条           |     |
| 纸条 X      |      (0, 0, 8) | 放在房间 B 中部偏下         |     |
| 门 B       |    (3.5, 0, 4) | 房间 B 与房间 A 之间       |     |
| 柜子 C      | (-3.5, 0, 0.5) | 房间 A 左侧，和门 A 形成回头路线 |     |
| 门 A       | (3.5, 0, -3.5) | 房间 A 与户外之间          |     |
| 终点触发区 END |     (0, 0, -8) | 户外区域中央偏下            |     |

> [!note] 坐标不是硬性要求
> 坐标的价值是帮助实现者快速摆出第一版白盒。只要空间关系、路线和互动逻辑保持一致，实际坐标可以调整。

## 6. 区域设计

### 6.1 房间 B

功能：

- 玩家出生。
- 玩家学习移动。
- 玩家发现纸条 X。
- 玩家根据纸条提示寻找门 B。

房间 B 需要包含：

| 元素 | 位置建议 | 设计目的 |
|---|---|---|
| 玩家出生点 | 房间左上或中上 | 给玩家一点移动空间 |
| 纸条 X | 房间中央偏下 | 玩家稍微探索即可看到 |
| 门 B | 房间右下或下方 | 作为第一个出口目标 |
| 墙体 | 四周 | 阻挡玩家离开边界 |

摆放原则：

- 玩家出生后不要直接站在纸条上，要走几步才能发现。
- 纸条 X 不要贴墙，避免玩家错过。
- 门 B 应该在玩家读完纸条后容易注意到。
- 房间 B 不放柜子，避免玩家误以为钥匙在起点房间。

### 6.2 房间 A

功能：

- 玩家从门 B 进入。
- 玩家第一次尝试门 A，发现上锁。
- 玩家探索柜子，获得钥匙。
- 玩家返回门 A，用钥匙离开。

房间 A 需要包含：

| 元素 | 位置建议 | 设计目的 |
|---|---|---|
| 门 B 入口 | 房间上方或右上 | 玩家从这里进入 |
| 门 A | 房间下方或右下 | 终点出口，先锁住 |
| 柜子 C | 房间左侧或左下 | 让玩家在锁门后回头探索 |
| 墙体 | 四周 | 阻挡玩家离开边界 |

摆放原则：

- 门 A 应该比柜子更容易先被玩家看到。
- 玩家先摸到门 A，看到“门上锁了”，再找柜子，这样钥匙逻辑更清楚。
- 柜子不要藏在角落阴影里，第一版不是找茬游戏。
- 柜子和门 A 之间保持 3 到 6 秒移动距离，让“拿钥匙回去开门”成立。

### 6.3 户外区域

功能：

- 玩家通过门 A 后进入。
- 玩家走到终点触发区。
- Demo 显示结束提示。

户外区域需要包含：

| 元素 | 位置建议 | 设计目的 |
|---|---|---|
| 门 A 出口 | 户外上方 | 玩家从室内出来的位置 |
| 终点触发区 END | 户外中央偏下 | 触发 Demo 结束 |
| 简单边界 | 四周或自然边界 | 防止玩家走出测试地图 |

摆放原则：

- 户外不做大地图，只做一个“离开小屋”的结束空间。
- 终点触发区可以用亮色地面、出口牌、光柱或简单标记表示。
- 玩家进入户外后 3 到 6 秒内应能触发 Demo 结束。

## 7. 玩家路线

### 7.1 标准路线

```text
出生点 P
  -> 纸条 X
  -> 门 B
  -> 房间 A
  -> 门 A
  -> 提示“门上锁了。”
  -> 柜子 C
  -> 获得钥匙
  -> 返回门 A
  -> 门 A 打开
  -> 户外区域
  -> 终点 END
  -> Demo 结束
```

### 7.2 允许的非标准路线

玩家不一定按完美顺序行动，以下情况需要被允许：

| 玩家行为 | 结果 |
|---|---|
| 不读纸条直接摸门 B | 可以提示“门像是能打开，但你还没有想好要去哪。”，也可以简化为允许打开 |
| 进入房间 A 后先摸柜子 | 获得钥匙 |
| 拿到钥匙前摸门 A | 显示“门上锁了。” |
| 拿到钥匙后摸门 A | 显示“试着用刚才的钥匙，结果门打开了。” |
| 反复摸柜子 | 第一次给钥匙，之后显示“柜子是空的。” |
| 反复摸已经打开的门 | 不阻挡玩家，可不再显示文本 |

## 8. 互动对象摆放规格

| 对象 | 区域 | 可见性要求 | 互动半径 | 状态 |
|---|---|---|---:|---|
| 纸条 X | 房间 B | 从出生点移动几步后可见 | 1.5 到 2 | 可重复阅读 |
| 门 B | 房间 B / A 之间 | 房间 B 内应明显可见 | 1.5 到 2 | 关闭 -> 打开 |
| 柜子 C | 房间 A | 进入房间 A 后转向即可看到 | 1.5 到 2 | 未搜索 -> 已搜索 |
| 门 A | 房间 A / 户外之间 | 房间 A 内应明显可见 | 1.5 到 2 | 上锁 -> 打开 |
| 终点 END | 户外 | 通过门 A 后应明显可见 | 2 到 3 | 未触发 -> 已触发 |

## 9. 碰撞与通行规则

| 元素 | 初始碰撞 | 状态变化 |
|---|---|---|
| 房间墙体 | 阻挡玩家 | 不变化 |
| 门 B | 阻挡玩家 | 打开后不再阻挡 |
| 门 A | 阻挡玩家 | 有钥匙并打开后不再阻挡 |
| 柜子 | 可阻挡，也可不阻挡 | 不影响主流程 |
| 终点触发区 | 不阻挡玩家 | 玩家进入后触发 Demo 结束 |

## 10. 镜头检查

由于 Demo 使用固定角度、不可旋转、始终跟随玩家的上帝视角 / 俯视角镜头，白盒必须保证在这个固定角度下能看清互动对象。

注意：这里不是越肩第三人称，不是贴近角色背后的肩后镜头。镜头必须能看见角色周围环境，方便玩家判断路线、房间结构和可互动物体。

镜头参考：

- 《极乐迪斯科》：固定角度探索镜头，玩家不旋转镜头。
- 《孤胆枪手》：固定角度跟随角色，角色移动时画面稳定跟随。

镜头规则：

| 项目 | 要求 |
|---|---|
| 角度 | 固定 |
| 旋转 | 玩家不能旋转 |
| 跟随 | 始终跟随玩家角色 |
| 是否越肩 | 不是越肩视角 |
| 用途 | 保证路线、门、柜子、纸条和终点可读 |

检查要求：

- 玩家出生时能看见房间 B 的大部分空间。
- 纸条 X 不被墙体、门框、柜子或 UI 遮挡。
- 门 B 和门 A 的位置清楚。
- 房间 A 中，玩家从门 B 进入后，能看见门 A 或至少能自然走向门 A。
- 柜子不能被上方墙体遮住。
- 户外终点标记不能出屏。

由于镜头固定不旋转，建议所有关键对象都放在墙体内侧，不要贴在被镜头遮挡的上墙位置。

## 11. 白盒美术标识

第一版可用纯色区分对象，方便测试：

| 对象 | 建议颜色 |
|---|---|
| 地面 | 灰色 |
| 墙体 | 深灰色 |
| 门 B | 蓝色 |
| 门 A | 红色或橙色 |
| 纸条 X | 白色 |
| 柜子 | 棕色 |
| 终点触发区 | 绿色或亮黄色 |
| 玩家 | 明亮颜色，如青色 |

颜色只用于测试，不代表最终美术。

## 12. 白盒验收清单

### 12.1 空间验收

- [ ] 房间 B、房间 A、户外区域都已经搭建。
- [ ] 房间 B 与房间 A 只通过门 B 相连。
- [ ] 房间 A 与户外区域只通过门 A 相连。
- [ ] 玩家出生在房间 B。
- [ ] 纸条 X 位于房间 B。
- [ ] 柜子位于房间 A。
- [ ] 门 A 位于房间 A 与户外之间。
- [ ] 户外区域有终点触发区。

### 12.2 路线验收

- [ ] 玩家从出生点能自然走到纸条 X。
- [ ] 玩家从纸条 X 能自然走到门 B。
- [ ] 玩家打开门 B 后能进入房间 A。
- [ ] 玩家在房间 A 能看到或找到门 A。
- [ ] 玩家在房间 A 能看到或找到柜子。
- [ ] 玩家从柜子返回门 A 的路线清楚。
- [ ] 玩家打开门 A 后能进入户外区域。
- [ ] 玩家进入户外后能自然走到终点触发区。

### 12.3 可读性验收

- [ ] 纸条、门、柜子、终点在俯视角下都能被识别。
- [ ] 玩家不能旋转镜头。
- [ ] 镜头始终跟随玩家角色。
- [ ] 镜头不是越肩视角，能看见玩家周围环境。
- [ ] 镜头跟随过程中不会把玩家或关键对象甩出画面。
- [ ] 墙体不会遮挡关键对象。
- [ ] 门的打开/关闭状态有明显差异。
- [ ] 柜子和普通装饰物能区分。
- [ ] 终点区域能让玩家理解“这里是出口”。

## 13. 交付给实现者的白盒任务

实现者需要完成：

1. 按本文档搭建一个由房间 B、房间 A、户外组成的小地图。
2. 在房间 B 放置玩家出生点和纸条 X。
3. 在房间 B 与房间 A 之间放置门 B。
4. 在房间 A 放置柜子。
5. 在房间 A 与户外之间放置门 A。
6. 在户外放置终点触发区。
7. 设置墙体和关闭门的碰撞。
8. 设置固定角度、不可旋转、始终跟随玩家的镜头。
9. 确保镜头下所有关键对象清楚可见。
10. 用简单颜色或形状区分关键对象。

## 14. 后续可扩展方向

白盒验证成功后，可以继续往三个方向扩展：

| 扩展方向 | 内容 |
|---|---|
| 氛围版 | 加入废弃小屋的低多边形模型、昏暗光照、旧家具 |
| 叙事版 | 将纸条替换成“会员名册碎页”或儿童组织提示 |
| 42号楼版 | 将废弃小屋结构替换为 42号楼的一层和二层局部 |

当前阶段先不要扩展，先确认最小流程跑通。
