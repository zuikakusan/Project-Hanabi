# 最小 Demo 测试验收表

## 1. 文档目的

这份文档用于在实现者交付废弃小屋 Demo 后，由策划逐条验收。

验收重点不是画面好不好看，而是确认：

- 玩家能否完整走通流程。
- 移动、碰撞、互动、文本、钥匙、门和终点是否按设计运行。
- 玩家乱序操作时，Demo 不会卡死。
- 第一版是否达到“可玩最小闭环”。

## 2. 验收结论标准

### 2.1 通过标准

Demo 可以判定为“通过第一版验收”，需要满足：

- P0 测试项全部通过。
- 没有会阻止玩家完成 Demo 的卡死问题。
- 玩家可以从房间 B 出发，经过房间 A，拿到钥匙，打开门 A，到达户外并触发结束。

### 2.2 问题等级

| 等级 | 定义 | 示例 |
|---|---|---|
| P0 | 阻断主流程，必须修 | 玩家无法打开门 A，无法结束 Demo |
| P1 | 明显影响体验，应尽快修 | 提示文本错误，玩家容易误解 |
| P2 | 小问题，可后续修 | 颜色不够好看，装饰物穿模 |

### 2.3 验收记录格式

建议每次测试记录：

| 阻断问题 |  |
| 备注 |  |

## 3. 测试前准备

测试前确认：

- 可以启动 Demo。
- 有一个主场景。
- 玩家出生在房间 B。
- 地图中包含房间 B、房间 A、户外区域。
- 地图中包含纸条 X、门 B、柜子、门 A、户外终点触发区。
- 操作方式为 W / A / S / D 移动，E 互动。

## 4. P0 主流程测试

### TC-P0-001 正常通关流程

| 项目 | 内容 |
|---|---|
| 优先级 | P0 |
| 前置条件 | 从新游戏或重启 Demo 开始 |
| 测试目标 | 验证玩家能完整走通标准流程 |

步骤：

1. 进入 Demo。
2. 确认玩家出生在房间 B。
3. 移动到纸条 X 附近。
4. 按 E 阅读纸条。
5. 移动到门 B 附近。
6. 按 E 打开门 B。
7. 进入房间 A。
8. 移动到门 A 附近。
9. 按 E 互动门 A。
10. 移动到柜子附近。
11. 按 E 搜索柜子。
12. 返回门 A。
13. 按 E 打开门 A。
14. 进入户外区域。
15. 走到终点触发区。

预期结果：

- 纸条显示“纸条上写着：离开房间。”
- 门 B 可以打开。
- 没有钥匙时门 A 显示“门上锁了。”
- 柜子显示“柜子里有一把钥匙。”
- 获得钥匙后门 A 显示“试着用刚才的钥匙，结果门打开了。”
- 门 A 打开后可以进入户外。
- 到达户外终点后显示“Demo 结束。”

通过标准：

- 玩家能在 1 到 3 分钟内完成完整流程。
- 过程中没有卡死、穿墙、无法互动或文本无法关闭的问题。

### TC-P0-002 玩家移动与碰撞

| 项目 | 内容 |
|---|---|
| 优先级 | P0 |
| 前置条件 | 进入 Demo |
| 测试目标 | 验证移动和基础阻挡 |

步骤：

1. 使用 W / A / S / D 向八个方向移动。
2. 尝试走向房间墙体。
3. 尝试穿过未打开的门 B。
4. 尝试穿过未打开的门 A。

预期结果：

- 玩家可以顺畅移动。
- 玩家不能穿墙。
- 未打开的门 B 会阻挡玩家。
- 未打开的门 A 会阻挡玩家。
- 玩家停止移动时停在原地，不滑动。

通过标准：

- 移动清楚可控。
- 墙体和关闭的门不会被穿过。

### TC-P0-003 门 A 钥匙逻辑

| 项目 | 内容 |
|---|---|
| 优先级 | P0 |
| 前置条件 | 玩家已进入房间 A，但还没有搜索柜子 |
| 测试目标 | 验证门 A 必须有钥匙才能打开 |

步骤：

1. 靠近门 A。
2. 按 E。
3. 确认门 A 没有打开。
4. 去柜子处按 E。
5. 返回门 A。
6. 再次按 E。

预期结果：

- 第一次互动门 A 显示“门上锁了。”
- 第一次互动后门 A 仍然阻挡玩家。
- 搜索柜子后获得钥匙。
- 第二次互动门 A 显示“试着用刚才的钥匙，结果门打开了。”
- 门 A 打开后不再阻挡玩家。

通过标准：

- 没有钥匙不能通过门 A。
- 有钥匙可以打开并通过门 A。

## 5. P0 互动对象测试

### TC-P0-004 纸条 X

| 项目 | 内容 |
|---|---|
| 优先级 | P0 |
| 前置条件 | 玩家在房间 B |
| 测试目标 | 验证纸条互动 |

步骤：

1. 靠近纸条 X。
2. 确认出现互动提示。
3. 按 E。
4. 关闭文本。
5. 再次按 E 阅读纸条。

预期结果：

- 靠近纸条时出现 `[E]` 或 `按 E 互动`。
- 文本显示“纸条上写着：离开房间。”
- 文本可以关闭。
- 纸条可以重复阅读。

### TC-P0-005 门 B

| 项目 | 内容 |
|---|---|
| 优先级 | P0 |
| 前置条件 | 玩家在房间 B |
| 测试目标 | 验证门 B 互动 |

步骤：

1. 不读纸条，直接靠近门 B。
2. 按 E。
3. 阅读纸条 X。
4. 再次靠近门 B。
5. 按 E。
6. 尝试通过门 B。

预期结果：

- 未读纸条时，门 B 不打开，并显示“门像是能打开，但你还没有想好要去哪。”
- 读纸条后，门 B 显示“门打开了。”
- 门 B 打开后可以通过。

> [!note] 如果项目决定简化门 B
> 如果后续决定“门 B 不需要读纸条也能打开”，本条用例需要同步更新。当前验收表按“读纸条后才能打开”执行。

### TC-P0-006 柜子

| 项目 | 内容 |
|---|---|
| 优先级 | P0 |
| 前置条件 | 玩家已进入房间 A |
| 测试目标 | 验证柜子和钥匙状态 |

步骤：

1. 靠近柜子。
2. 确认出现互动提示。
3. 按 E。
4. 关闭文本。
5. 再次按 E。

预期结果：

- 第一次互动显示“柜子里有一把钥匙。”
- 第一次互动后玩家获得门 A 钥匙。
- 第二次互动显示“柜子是空的。”
- 第二次互动不会重复获得钥匙。

### TC-P0-007 户外终点

| 项目 | 内容 |
|---|---|
| 优先级 | P0 |
| 前置条件 | 门 A 已打开 |
| 测试目标 | 验证 Demo 结束 |

步骤：

1. 通过门 A 进入户外。
2. 走向终点触发区。
3. 进入触发区。

预期结果：

- 屏幕显示“Demo 结束。”
- Demo 结束提示只触发一次。
- 推荐：玩家移动被暂停，停留在结束画面。

## 6. P1 乱序与异常操作测试

### TC-P1-001 不读纸条直接找门 B

| 项目 | 内容 |
|---|---|
| 优先级 | P1 |
| 前置条件 | 玩家刚出生，未读纸条 |
| 测试目标 | 验证错误顺序反馈 |

步骤：

1. 出生后不读纸条。
2. 直接走到门 B。
3. 按 E。

预期结果：

- 门 B 不打开。
- 显示“门像是能打开，但你还没有想好要去哪。”
- 玩家仍可返回纸条 X，不会卡死。

### TC-P1-002 进入房间 A 后先搜柜子

| 项目 | 内容 |
|---|---|
| 优先级 | P1 |
| 前置条件 | 玩家已经进入房间 A，还没摸门 A |
| 测试目标 | 验证允许玩家提前获得钥匙 |

步骤：

1. 进入房间 A。
2. 不先互动门 A。
3. 直接靠近柜子。
4. 按 E 获得钥匙。
5. 再去门 A。
6. 按 E。

预期结果：

- 柜子正常给钥匙。
- 门 A 直接显示“试着用刚才的钥匙，结果门打开了。”
- 不强制玩家必须先看“门上锁了。”

### TC-P1-003 重复互动已打开的门

| 项目 | 内容 |
|---|---|
| 优先级 | P1 |
| 前置条件 | 门 B 或门 A 已经打开 |
| 测试目标 | 验证重复互动不会出错 |

步骤：

1. 打开门 B。
2. 再次靠近门 B 并按 E。
3. 打开门 A。
4. 再次靠近门 A 并按 E。

预期结果：

- 不会报错、卡死或重新关闭门。
- 可以无文本，或显示“门已经打开了。”
- 玩家仍可自由通过已经打开的门。

### TC-P1-004 文本显示时移动

| 项目 | 内容 |
|---|---|
| 优先级 | P1 |
| 前置条件 | 任意可互动对象附近 |
| 测试目标 | 验证文本框期间的控制状态 |

步骤：

1. 与纸条、柜子或门互动，弹出文本框。
2. 文本框显示期间尝试移动。
3. 关闭文本框后再次移动。

预期结果：

- 推荐：文本显示期间玩家不能移动。
- 关闭文本后玩家恢复移动。
- 不会因为移动导致重复触发互动。

## 7. P1 UI 与可读性测试

### TC-P1-005 互动提示显示与消失

| 项目 | 内容 |
|---|---|
| 优先级 | P1 |
| 前置条件 | 进入 Demo |
| 测试目标 | 验证互动提示清楚 |

步骤：

1. 靠近纸条 X。
2. 远离纸条 X。
3. 靠近门 B。
4. 远离门 B。
5. 靠近柜子和门 A。

预期结果：

- 靠近可互动对象时显示提示。
- 离开互动范围后提示消失。
- 提示不会一直残留在屏幕上。
- 提示不会遮挡关键文本。

### TC-P1-006 同时靠近多个对象

| 项目 | 内容 |
|---|---|
| 优先级 | P1 |
| 前置条件 | 玩家位于两个互动对象距离较近的位置 |
| 测试目标 | 验证互动对象选择规则 |

步骤：

1. 站到两个可互动对象附近。
2. 按 E。
3. 观察触发的是哪个对象。

预期结果：

- 优先触发距离玩家最近的对象。
- 不会同时触发两个对象。
- 不会出现两个文本框叠加。

## 8. P1 地图与镜头测试

### TC-P1-007 固定跟随镜头

| 项目 | 内容 |
|---|---|
| 优先级 | P1 |
| 前置条件 | 进入 Demo |
| 测试目标 | 验证镜头为固定角度、不可旋转、始终跟随玩家，且不是越肩视角 |

步骤：

1. 控制玩家在房间 B 内移动。
2. 尝试使用鼠标、键盘或其他输入旋转镜头。
3. 从房间 B 进入房间 A。
4. 从房间 A 进入户外区域。

预期结果：

- 镜头始终跟随玩家角色。
- 玩家不能旋转镜头。
- 镜头角度保持固定。
- 镜头不是越肩视角，能看见玩家周围环境。
- 玩家移动时不会离开镜头主要视野。
- 关键对象不会因为镜头固定而长期不可见。

### TC-P1-008 关键对象可见性

| 项目 | 内容 |
|---|---|
| 优先级 | P1 |
| 前置条件 | 进入 Demo |
| 测试目标 | 验证俯视角下关键对象能被识别 |

步骤：

1. 在出生点观察房间 B。
2. 在房间 B 内寻找纸条和门 B。
3. 进入房间 A 后寻找门 A 和柜子。
4. 进入户外后寻找终点触发区。

预期结果：

- 纸条 X 清楚可见。
- 门 B 和门 A 清楚可见。
- 柜子和普通装饰物能区分。
- 户外终点触发区清楚可见。
- 墙体不会遮挡关键对象。

### TC-P1-009 地图尺寸与流程时长

| 项目 | 内容 |
|---|---|
| 优先级 | P1 |
| 前置条件 | 从 Demo 开始 |
| 测试目标 | 验证地图不会过大或过小 |

步骤：

1. 按标准流程完成 Demo。
2. 记录完成时间。
3. 观察是否有明显空跑时间。

预期结果：

- 第一次测试完成时间建议在 1 到 3 分钟。
- 玩家不会长时间找不到纸条、柜子或门。
- 路线清楚，但不是出生点旁边一键通关。

## 9. P2 表现问题记录

以下内容不阻断第一版，但可以记录给后续优化：

- 角色动画是否僵硬。
- 门打开是否缺少表现。
- 柜子是否不够像柜子。
- 纸条是否太小。
- 户外终点是否太像普通地面。
- 文本框是否太简陋。
- 房间是否太空。
- 俯视角是否压迫感不足。
- 固定镜头角度是否导致个别物体不够清楚。

## 10. 总体验收清单

### 10.1 必须通过

- [ ] 玩家可以进入 Demo。
- [ ] 玩家可以移动。
- [ ] 玩家不能穿墙。
- [ ] 关闭的门会阻挡玩家。
- [ ] 纸条 X 可以阅读。
- [ ] 门 B 可以按规则打开。
- [ ] 玩家可以进入房间 A。
- [ ] 无钥匙时门 A 上锁。
- [ ] 柜子可以获得钥匙。
- [ ] 有钥匙时门 A 可以打开。
- [ ] 玩家可以进入户外。
- [ ] 户外终点可以触发“Demo 结束。”
- [ ] 整体流程无卡死。

### 10.2 建议通过

- [ ] 所有互动对象靠近时都有提示。
- [ ] 文本框可以正常关闭。
- [ ] 文本显示期间玩家不会误移动。
- [ ] 重复互动不会产生错误。
- [ ] 关键对象在俯视角下清楚可见。
- [ ] 镜头固定角度，不可旋转。
- [ ] 镜头始终跟随玩家角色。
- [ ] 镜头不是越肩视角，能看见玩家周围环境。
- [ ] 玩家能在 1 到 3 分钟内完成 Demo。
