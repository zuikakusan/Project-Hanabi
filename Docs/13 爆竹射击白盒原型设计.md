# 爆竹射击白盒原型设计

## 1. 文档目的

这份文档用于设计《千禧年战纪》的第一版爆竹射击白盒原型。

当前目标不是做完整战斗系统，而是验证固定上帝视角下的几个基础手感：

- 玩家能否用键盘移动。
- 玩家能否用鼠标控制人物朝向。
- 固定上帝视角下，鼠标瞄准是否清楚。
- 左键投掷是否成立。
- 右键点火是否成立。
- 摔炮落地即爆是否清楚。
- 擦炮点火后 5 秒爆炸是否有趣。
- 靶子受击反馈是否清楚。

## 2. 原型定位

原型名称：

> 爆竹射击白盒靶场

一句话目标：

> 在一个小型白盒靶场中，验证固定上帝视角下“键盘移动 + 鼠标控制朝向 + 右键点火 + 左键投掷”的爆竹射击手感。

## 3. 当前不接入的内容

本原型暂时不接入废弃小屋 Demo，也不接入正式秘境。

暂时不做：

- 敌人 AI
- 敌人攻击玩家
- 玩家受伤
- 血量系统
- 背包系统
- 商店系统
- 掉落奖励
- 任务系统
- 复杂关卡
- Boss
- 魔术棒
- 窜天猴
- 噪音吸引系统

第一版只做一个干净的射击测试场。

## 4. 视角与镜头

### 4.1 镜头规则

| 项目 | 规则 |
|---|---|
| 镜头类型 | 固定上帝视角 / 斜俯视第三人称 |
| 是否越肩 | 不是越肩视角 |
| 是否可旋转 | 不可旋转 |
| 是否跟随玩家 | 始终跟随玩家 |
| 可见范围 | 必须能看见玩家周围环境、靶子、爆炸范围和障碍物 |
| 参考游戏 | 《孤胆枪手》 |

### 4.2 镜头验收

- 玩家移动时，镜头始终跟随玩家。
- 鼠标移动只控制人物朝向，不控制镜头旋转。
- 玩家不能旋转镜头。
- 镜头需要能看见玩家周围环境。
- 投掷方向、靶子位置、爆炸范围都要清楚。

## 5. 玩家控制

### 5.1 输入总表

| 输入 | 功能 |
|---|---|
| W / A / S / D | 玩家移动 |
| 鼠标移动 | 控制人物朝向 / 瞄准方向 |
| 鼠标左键 | 投掷当前武器 |
| 鼠标右键 | 点火 / 尝试点火 |
| 数字键 1 | 切换摔炮 |
| 数字键 2 | 切换擦炮 |
| Esc | 暂停或退出，可简化 |

### 5.2 移动与朝向关系

本原型要验证一个核心控制方式：

> 移动由键盘控制，人物朝向由鼠标控制。

这意味着：

- 玩家可以向一个方向移动，同时面朝另一个方向。
- 人物身体朝向始终指向鼠标在地面上的目标点。
- 鼠标移动时，人物朝向实时更新。
- WASD 不直接决定人物朝向。
- 停止移动后，人物仍保持朝向鼠标方向。

示例：

```text
玩家按 A 向左移动，但鼠标在右侧。
角色身体应该朝右，移动方向向左。
```

这个设计对后续射击很重要，因为玩家需要边撤退边朝敌人投掷爆竹。

## 6. 人物朝向系统

### 6.1 设计目标

人物朝向系统是本原型的 P0 功能。

它要验证：

- 固定上帝视角下，鼠标能否准确控制人物朝向。
- 人物朝向是否能作为投掷方向。
- 玩家是否能自然理解“鼠标指哪，人物朝哪”。

### 6.2 基础规则

| 项目 | 规则 |
|---|---|
| 朝向来源 | 鼠标指向的地面位置 |
| 朝向更新 | 实时更新 |
| 移动影响朝向 | 不影响 |
| 停止移动时 | 继续面向鼠标方向 |
| 投掷方向 | 使用人物当前朝向 |
| 点火方向需求 | 点火不改变朝向，但人物仍面向鼠标 |

### 6.3 表现要求

第一版可以用简单占位表现：

- 角色模型前方加一个明显的朝向标记。
- 可以使用箭头、鼻子方向、手臂方向或身体朝向。
- 鼠标方向最好有一个地面瞄准点。
- 不要求复杂转身动画。

推荐最低表现：

```text
角色身体有明显正面 + 地面有鼠标瞄准点
```

### 6.4 手感要求

| 项目 | 要求 |
|---|---|
| 转向响应 | 鼠标移动后朝向立刻或平滑跟随 |
| 转向速度 | 第一版可以瞬时转向，也可以快速平滑转向 |
| 抖动 | 鼠标静止时角色朝向不能乱抖 |
| 近距离鼠标 | 鼠标离角色很近时不要疯狂旋转 |

### 6.5 待确认

| 问题 | 当前建议 |
|---|---|
| 角色转向是瞬时还是平滑 | 第一版用快速平滑转向，手感更自然 |
| 鼠标离角色太近时如何处理 | 设置最小瞄准距离，小于该距离保持上一次朝向 |
| 是否显示瞄准线 | 第一版可以显示淡色方向线，方便调试 |

## 7. 射击靶场地图

### 7.1 地图目标

射击靶场只用于验证射击手感，不承担叙事。

地图需要：

- 小型空地。
- 清晰边界。
- 若干静止靶子。
- 少量障碍物。
- 玩家出生点。
- 武器测试空间。

### 7.2 地图结构示意

```text
┌──────────────────────────────┐
│                              │
│        T1        T2          │
│                              │
│              O               │
│                              │
│        P                     │
│                              │
│        T3              T4    │
│                              │
└──────────────────────────────┘
```

图例：

| 标记 | 含义 |
|---|---|
| P | 玩家出生点 |
| T1-T4 | 静止靶子 |
| O | 简单障碍物 |

### 7.3 尺寸建议

| 项目 | 建议 |
|---|---|
| 地图大小 | 20 x 20 到 30 x 30 单位 |
| 玩家到近靶距离 | 5 到 8 单位 |
| 玩家到远靶距离 | 12 到 18 单位 |
| 靶子数量 | 4 个 |
| 障碍物数量 | 1 到 3 个 |

### 7.4 靶子摆放

| 靶子 | 位置 | 用途 |
|---|---|---|
| T1 | 玩家前方近距离 | 测试基础投掷 |
| T2 | 玩家前方远距离 | 测试投掷距离 |
| T3 | 玩家侧后方 | 测试鼠标转向 |
| T4 | 障碍物后方 | 测试抛物线和爆炸范围 |

## 8. 靶子系统

### 8.1 靶子规则

第一版靶子不需要 AI。

靶子需要：

- 可以被爆炸命中。
- 命中后有反馈。
- 可以记录命中。
- 可以消失、变色或显示“命中”。

### 8.2 靶子状态

| 状态 | 说明 |
|---|---|
| 未命中 | 默认状态 |
| 被命中 | 被爆炸范围覆盖 |
| 已击倒 / 已消失 | 可选表现 |

### 8.3 命中反馈

第一版至少实现一种：

| 反馈 | 推荐 |
|---|---|
| 靶子变色 | 推荐 |
| 靶子消失 | 推荐 |
| 弹出“命中”文字 | 可选 |
| 粒子效果 | 可选 |
| 音效 | 可选 |

## 9. 武器一：摔炮

### 9.1 设计目的

摔炮用于验证最基础的投掷和爆炸。

### 9.2 操作流程

```text
切换到摔炮
  -> 鼠标控制人物朝向
  -> 左键投掷
  -> 摔炮沿人物朝向抛出
  -> 落地立刻爆炸
  -> 爆炸范围内靶子被命中
```

### 9.3 规则表

| 项目 | 规则 |
|---|---|
| 是否需要点火 | 不需要 |
| 点火输入 | 无 |
| 投掷输入 | 鼠标左键 |
| 投掷方向 | 人物当前朝向 |
| 弹道 | 抛物线 |
| 爆炸时机 | 落地立刻爆炸 |
| 爆炸范围 | 小范围 |
| 右键行为 | 显示提示“摔炮不用点。”或无反应 |

### 9.4 原型数值建议

| 参数 | 建议值 |
|---|---:|
| 投掷距离 | 6 到 8 单位 |
| 爆炸半径 | 1.5 到 2 单位 |
| 冷却时间 | 0.5 到 1 秒 |
| 弹药 | 无限，方便测试 |

## 10. 武器二：擦炮

### 10.1 设计目的

擦炮用于验证“点火 + 延迟爆炸 + 投掷预判”。

### 10.2 操作流程

```text
切换到擦炮
  -> 鼠标控制人物朝向
  -> 右键点火
  -> 点火成功
  -> 5 秒倒计时开始
  -> 左键投掷
  -> 擦炮沿人物朝向抛出
  -> 倒计时结束
  -> 爆炸
  -> 爆炸范围内靶子被命中
```

### 10.3 规则表

| 项目 | 规则 |
|---|---|
| 是否需要点火 | 需要 |
| 点火输入 | 鼠标右键 |
| 投掷输入 | 鼠标左键 |
| 投掷方向 | 人物当前朝向 |
| 爆炸计时起点 | 点火成功时 |
| 爆炸延迟 | 5 秒 |
| 弹道 | 抛物线 |
| 爆炸范围 | 中等范围 |
| 未点火能否投掷 | 第一版不允许 |
| 点燃后能否取消 | 第一版不允许 |

### 10.4 点火方式

第一版可以先只做一种点火：

| 点火方式 | 当前建议 |
|---|---|
| 打火机点火 | 必定成功 |

摩擦片点火失败率可以后续再做。第一版先验证点火状态和 5 秒倒计时。

### 10.5 原型数值建议

| 参数 | 建议值 |
|---|---:|
| 投掷距离 | 7 到 10 单位 |
| 爆炸半径 | 2.5 到 3 单位 |
| 爆炸延迟 | 5 秒 |
| 冷却时间 | 1 秒 |
| 弹药 | 无限，方便测试 |

### 10.6 已点燃但未投掷

第一版建议：

| 情况 | 处理 |
|---|---|
| 点火后未投掷，倒计时结束 | 在玩家当前位置爆炸，或自动掉落爆炸 |
| 是否伤害玩家 | 第一版不伤害玩家，只做视觉反馈 |

这样可以验证“点火后不能拖太久”的节奏，但不引入玩家受伤系统。

## 11. UI 与反馈

### 11.1 必须 UI

| UI | 用途 |
|---|---|
| 当前武器名称 | 显示摔炮 / 擦炮 |
| 点火状态 | 显示未点火 / 已点燃 |
| 擦炮倒计时 | 显示 5 秒倒计时 |
| 简单准星 | 显示鼠标瞄准位置 |
| 命中反馈 | 显示靶子被命中 |

### 11.2 推荐文本

| 情况 | 文本 |
|---|---|
| 切换摔炮 | 当前武器：摔炮 |
| 切换擦炮 | 当前武器：擦炮 |
| 摔炮右键 | 摔炮不用点。 |
| 擦炮未点火左键 | 还没点着。 |
| 擦炮点火成功 | 点着了。 |
| 擦炮倒计时 | 5 / 4 / 3 / 2 / 1 |
| 命中靶子 | 命中 |

## 12. 视觉与特效

第一版只需要占位表现：

| 对象 | 占位表现 |
|---|---|
| 玩家 | 简单模型，有明确正面 |
| 朝向 | 箭头、手臂方向或身体朝向 |
| 准星 | 地面小圆点 |
| 摔炮 | 小球或小方块 |
| 擦炮 | 小红色/灰色方块 |
| 爆炸 | 简单圆形范围、闪光或粒子 |
| 靶子 | 柱子、方块或假人 |

爆炸范围必须可读。美术可以很粗，但玩家要知道哪里炸了。

## 13. 验收清单

### 13.1 P0 必须通过

- [ ] 玩家可以进入射击靶场。
- [ ] 镜头固定、不可旋转、始终跟随玩家。
- [ ] 玩家可以用 W / A / S / D 移动。
- [ ] 鼠标移动可以控制人物朝向。
- [ ] 人物朝向不由移动方向决定。
- [ ] 玩家可以边移动边朝鼠标方向瞄准。
- [ ] 数字键 1 可以切换摔炮。
- [ ] 数字键 2 可以切换擦炮。
- [ ] 摔炮左键投掷。
- [ ] 摔炮落地立刻爆炸。
- [ ] 摔炮能命中靶子。
- [ ] 擦炮右键点火。
- [ ] 擦炮点火后开始 5 秒倒计时。
- [ ] 擦炮左键投掷。
- [ ] 擦炮倒计时结束后爆炸。
- [ ] 擦炮能命中靶子。
- [ ] 靶子被命中后有反馈。

### 13.2 P1 建议通过

- [ ] 鼠标准星清楚。
- [ ] 人物正面清楚。
- [ ] 投掷方向和人物朝向一致。
- [ ] 爆炸范围清楚。
- [ ] 擦炮倒计时 UI 清楚。
- [ ] 摔炮和擦炮手感有明显差异。
- [ ] 靶子距离能测试近距离和远距离投掷。
- [ ] 障碍物后方靶子可以测试抛物线投掷。

## 14. 当前待确认问题

| 问题 | 当前建议 |
|---|---|
| 人物转向是瞬时还是平滑 | 快速平滑转向 |
| 鼠标离角色太近时如何处理 | 保持上一朝向 |
| 是否显示方向线 | 第一版可显示淡色方向线，方便调试 |
| 擦炮点火是否会失败 | 第一版先不失败 |
| 擦炮炸在玩家脚下是否伤害玩家 | 第一版不伤害 |
| 靶子是否需要血量 | 第一版不需要，命中即反馈 |

## 15. 交付给实现者的最小任务

实现者第一版只需要完成：

1. 一个可运行的射击靶场场景。
2. 固定上帝视角 / 斜俯视镜头。
3. WASD 移动。
4. 鼠标控制人物朝向。
5. 当前武器切换：摔炮、擦炮。
6. 摔炮左键投掷、落地爆炸、命中靶子。
7. 擦炮右键点火、5 秒倒计时、左键投掷、倒计时结束爆炸。
8. 靶子命中反馈。
9. 简单 UI：当前武器、点火状态、倒计时。
