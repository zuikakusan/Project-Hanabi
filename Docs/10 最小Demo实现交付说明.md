# 最小 Demo 实现交付说明

## 1. 本次交付目标

本次目标是实现一个可以运行的 3D 上帝视角探索 Demo。

Demo 场景为“废弃小屋”，由房间 B、房间 A 和户外区域组成。玩家需要从房间 B 出发，阅读纸条，打开门 B，进入房间 A，发现门 A 上锁，搜索柜子获得钥匙，再用钥匙打开门 A，进入户外并触发 Demo 结束。

一句话目标：

> 做出一个能完整跑通“探索 -> 互动 -> 获得信息 -> 找钥匙 -> 开门 -> 结束”的最小可玩版本。

## 3. 必须完成内容

### 3.1 基础运行

- 可以启动 Demo。
- 进入后加载主场景。
- 玩家出生在房间 B。
- 玩家可以使用 W / A / S / D 移动。
- 玩家可以使用 E 互动。
- Esc 可以暂停或退出，第一版可简化。

### 3.2 玩家控制

必须实现：

- 八方向移动。
- 面向移动方向。
- 停止移动时保持站立。
- 不能穿墙。
- 不能穿过未打开的门。
- 能被墙体、关闭门和必要障碍物阻挡。

### 3.3 镜头

必须实现：

- 固定角度镜头。
- 镜头不可由玩家旋转。
- 镜头始终跟随玩家角色。
- 镜头不是越肩视角，不是肩后 TPS。
- 镜头必须能看见玩家周围环境。
- 镜头参考《极乐迪斯科》和《孤胆枪手》的固定视角探索 / 射击阅读方式。
- 镜头应保证纸条、门 B、柜子、门 A 和户外终点等关键对象可见。

### 3.4 地图白盒

必须包含：

- 房间 B。
- 房间 A。
- 户外区域。
- 玩家出生点。
- 纸条 X。
- 门 B。
- 柜子。
- 门 A。
- 户外终点触发区。

地图结构：

```text
房间 B -- 门 B -- 房间 A -- 门 A -- 户外区域
                         |
                       柜子
```

### 3.5 互动对象

必须实现以下对象：

| 对象 | 必须功能 |
|---|---|
| 纸条 X | 靠近显示互动提示，按 E 显示文本 |
| 门 B | 读纸条后可以打开，打开后允许进入房间 A |
| 柜子 | 第一次互动获得门 A 钥匙，重复互动显示空 |
| 门 A | 无钥匙时上锁，有钥匙后打开 |
| 户外终点 | 玩家进入后显示 Demo 结束 |

### 3.6 UI

第一版只需要：

- 互动提示。
- 文本框。
- Demo 结束提示。

不需要复杂 UI。

## 4. 确定规则

本次实现请按以下规则执行：

| 规则 | 当前决定 |
|---|---|
| 镜头角度 | 固定角度 |
| 玩家是否能旋转镜头 | 不能 |
| 镜头是否跟随玩家 | 始终跟随 |
| 镜头是否越肩 | 不是越肩视角 |
| 镜头可见范围 | 必须能看见玩家周围环境 |
| 门 B 是否必须读纸条后才能打开 | 是 |
| 门 B 未读纸条时反馈 | “门像是能打开，但你还没有想好要去哪。” |
| 纸条 X 文本 | “纸条上写着：离开房间。” |
| 柜子第一次互动文本 | “柜子里有一把钥匙。” |
| 柜子重复互动文本 | “柜子是空的。” |
| 门 A 无钥匙文本 | “门上锁了。” |
| 门 A 有钥匙文本 | “试着用刚才的钥匙，结果门打开了。” |
| 终点文本 | “Demo 结束。” |
| 钥匙是否需要背包 UI | 不需要，只记录状态 |
| 钥匙是否消耗 | 不消耗 |
| 柜子是否需要动画 | 不需要 |
| 终点触发后 | 推荐停留结束画面，玩家不能继续移动 |

## 5. 状态变量需求

实现中至少需要记录以下状态：

| 状态变量 | 初始值 | 用途 |
|---|---|---|
| has_read_note_x | false | 是否读过纸条 X |
| door_b_open | false | 门 B 是否打开 |
| cabinet_a_searched | false | 柜子是否已经搜索过 |
| has_key_door_a | false | 是否获得门 A 钥匙 |
| door_a_open | false | 门 A 是否打开 |
| demo_finished | false | Demo 是否结束 |

变量命名可以调整，但逻辑必须一致。

## 6. 不要做的内容

第一版不要实现：

- 战斗系统。
- 敌人系统。
- 背包界面。
- 道具栏。
- 商店系统。
- 存档系统。
- 技能系统。
- 任务列表。
- 多地图。
- 正式剧情。
- 复杂过场演出。
- 正式低多边形美术。
- 音效系统。
- 复杂暂停菜单。

如果时间充裕，也不要优先加这些。第一版只验证最小探索闭环。

## 7. 美术与表现要求

第一版使用白盒或 Godot 基础图形即可。

建议颜色：

| 对象 | 建议颜色 |
|---|---|
| 地面 | 灰色 |
| 墙体 | 深灰色 |
| 门 B | 蓝色 |
| 门 A | 红色或橙色 |
| 纸条 X | 白色 |
| 柜子 | 棕色 |
| 户外终点 | 绿色或亮黄色 |
| 玩家 | 明亮颜色，如青色 |

目标是“看得懂”，不是“好看”。

## 8. 交付前自测要求

实现者交付前，请至少自测以下内容：

- 玩家能从房间 B 出生。
- 玩家能移动。
- 镜头固定角度，不能旋转。
- 镜头始终跟随玩家。
- 镜头不是越肩视角，能看见玩家周围环境。
- 玩家不能穿墙。
- 玩家不能穿过关闭的门。
- 纸条 X 可以阅读。
- 不读纸条时，门 B 不打开。
- 读纸条后，门 B 可以打开。
- 玩家能进入房间 A。
- 没钥匙时，门 A 显示“门上锁了。”
- 柜子第一次互动可以获得钥匙。
- 柜子重复互动显示“柜子是空的。”
- 有钥匙后，门 A 可以打开。
- 玩家能进入户外。
- 进入户外终点后显示“Demo 结束。”

完整验收按 [[09 最小Demo测试验收表]] 执行。

## 10. 策划验收方式

策划将按照 [[09 最小Demo测试验收表]] 逐条验收。

通过标准：

- P0 测试项全部通过。
- 玩家可以完整完成标准流程。
- 没有卡死点。
- 没有会阻断通关的碰撞或互动问题。

如果 P0 未通过，版本不进入体验讨论，先修主流程。

## 11. 本版本完成定义

当玩家可以完成以下流程时，本版本视为完成：

```text
启动 Demo
  -> 出生在房间 B
  -> 阅读纸条 X
  -> 打开门 B
  -> 进入房间 A
  -> 发现门 A 上锁
  -> 搜索柜子获得钥匙
  -> 用钥匙打开门 A
  -> 进入户外区域
  -> 触发 Demo 结束
```

## 12. 后续不在本次范围

本次完成后，后续才考虑：

- 替换为《千禧年战纪》正式场景。
- 将纸条改成会员名册碎页。
- 将废弃小屋改成 42号楼局部。
- 加入音效。
- 加入低多边形美术。
- 加入更多可调查物。
- 加入简单演出。
- 加入文本氛围。

不要在第一版实现时提前扩展。先把一个最小闭环做硬。
